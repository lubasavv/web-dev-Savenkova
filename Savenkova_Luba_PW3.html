What way of including internal links into the HTML page you would consider generally preferable: absolute or relative links? Describe use cases for choosing absolute links and choosing relative links

I think which way to include internal links depends on the specific task. For example, when working with external resources, it is better to use absolute links. In our lab works, I think it's better to use relative links. For example, when downloading an image, you can drop it into the html folder and then easily interpret it into code. This is precisely the advantage of relative links. They make the code more portable, I would say. Absolute links can be used to link to specific pages of a website or to work with content management systems. Relative links are the same they are used within 1 website and in testing work, for example, in which the domain may later change (another advantage of relative links)

Research and write about image formats used in web-development. What is the difference between JPG, GIF and PNG formats? What is WEBP image type? What is SVG?

JPG GIF PNG
use lossy compression(some image quality is sacrificed to reduce file size) lossless compression, preserving image quality without any loss during compression. lossless compression, similar to GIF, maintaining image quality without any loss
well-suited for photographs and images with complex color gradients supports animation, making it suitable for simple animated images and graphics supports transparency and alpha channels, making it ideal for images with transparent backgrounds
does not support transparency supports transparency, well-suited for images with sharp edges, text, or simple graphics that require high-quality rendering

WEBP is a modern image format that offers more efficient compression and smaller file sizes compared to JPG and PNG. It supports lossy, which makes it versatile for various types of images. WEBP images provide improved performance on websites.

SVG (Scalable Vector Graphics) is an XML—based vector image format that scales without loss of quality. That is why it is ideal for responsive web design.SVG supports animation, interactivity, and accessibility features, making it suitable for icons, logos, and illustrations. SVG files are lightweight, editable with code, and can be styled with CSS
